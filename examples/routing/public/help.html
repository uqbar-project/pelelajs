Pelela CLI Commands
Command Description Options Example
pelela init [projectName] Initialize a new Pelela project with the base template — pelela init my-app
pelela new <path/componentName> Create a new component in the project --no-css, -c --component pelela new src/components/Button
pelela rename <path/oldComponentName> <path/newComponentName> Rename an existing component — pelela rename src/components/Button src/components/ActionButton
Note: The CLI commands are built with TypeScript and Commander.js, providing a clean interface for project scaffolding and component management.
ViewModel
Method Description Example
initialize() Called automatically when the component mounts. Set up initial state, start observers, or perform async setup. initialize() { this.count = 0; }
Note: The initialize() method is optional. It runs once after the view model is instantiated and before the view renders.
Naming Conventions

Pelela follows these naming conventions across the framework:

Scope Case Example
File names kebab-case my-button.pelela.html, my-button.view-model.ts
Child component tags kebab-case <my-component>, <app-header>
ViewModel class & property names CamelCase MyViewModel, this.userName
Note: These conventions follow standard web platform practices: custom elements require kebab-case names, while TypeScript uses CamelCase for classes and identifiers.
Element Bindings
Binding Valid Tags Description Example
bind-content Any element Bind element's text content to a view model property <div bind-content="message"></div>
bind-value <input>, <textarea>, <select> Bind element's value to a view model property <input bind-value="count" />
bind-src <img> Bind element's src attribute to a view model property <img bind-src="src" />
bind-alt <img> Bind element's alt attribute to a view model property <img bind-alt="description" />
bind-class Any element Bind element's class attribute to a view model property <div bind-class="activeClass" />
bind-style Any element Bind element's style object to a view model property <div bind-style="dynamicStyle" />
bind-enabled <input>, <select>, <button>, <textarea>, <optgroup>, <option>, <fieldset> Bind element's enabled state to a view model property <button bind-enabled="canSubmit" />
if Any element Conditionally show/hide element based on view model property <div if="visible">Content</div>
for-each Any element Render a collection of items using a template <div for-each="items">{{item}}</div>
click Any element Handle click events on elements <button click="handleClick">Click</button>
enter <input> Handle Enter key press events <input enter="submitForm" />
Parent / Child Components

Pelela provides three attribute prefixes to communicate between parent and child components:

Prefix Direction Description Example
prop-* Parent → Child Pass reactive property values <my-component prop-name="value">
link-* Parent ↔ Child Two-way binding between parent and child <my-component link-name="name">
const-* Parent → Child Pass constant values (evaluated once, not reactive) <my-component const-name="'John'">
<pelela view-model="Parent"> <my-component prop-name="value" link-name="name" const-label="'Hello'"></my-component> </pelela>
Important: All parent-child bindings must be used within a <pelela> or <component> root element, or they'll be rejected as security violations.
